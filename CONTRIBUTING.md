# Contributing to Privacy Search Browser

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## üöÄ Quick Start

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/privacy-search-browser.git`
3. Create a branch: `git checkout -b feature/my-feature`
4. Make your changes
5. Test: `node server.js`
6. Commit: `git commit -m 'Add my feature'`
7. Push: `git push origin feature/my-feature`
8. Open a Pull Request

## üìù Code Guidelines

### Style
- Use 4 spaces for indentation
- Keep functions small and focused
- Add comments for complex logic
- Use meaningful variable names

### Testing
- Test your changes locally
- Ensure syntax is valid: `node -c server.js`
- Test search functionality
- Test proxy functionality
- Test on different browsers

### Commits
- Use clear, descriptive commit messages
- Start with a verb: "Add", "Fix", "Update", "Remove"
- Keep commits atomic (one feature/fix per commit)

Example:
```
Add bookmark functionality
Fix search parsing for special characters
Update user-agent pool with latest browsers
```

## üêõ Bug Reports

When reporting bugs, please include:

1. **Description**: Clear description of the bug
2. **Steps to Reproduce**: How to trigger the bug
3. **Expected Behavior**: What should happen
4. **Actual Behavior**: What actually happens
5. **Environment**: Node.js version, OS, browser
6. **Screenshots**: If applicable
